Samsung Pay SDK 개발자 온보딩 가이드: CardManager를 활용한 카드 관리

1.0 소개

이 가이드는 Samsung Pay SDK를 처음 사용하는 개발자를 위해 작성되었습니다. 이 문서의 목표는 CardManager 클래스를 중심으로 삼성 페이의 핵심 카드 관리 기능(카드 추가, 조회, 인증)을 구현하는 실용적인 절차를 단계별로 안내하는 것입니다.

본 가이드는 신규 개발자가 Samsung Pay SDK의 카드 관리 아키텍처와 주요 API 사용법을 신속하게 파악하고, 실제 프로젝트에 기여할 수 있도록 돕는 것을 최우선 목표로 삼고 있습니다. 각 섹션은 이론적인 설명과 구체적인 코드 예제를 결합하여, 개발자가 SDK의 작동 방식을 명확히 이해하고 즉시 활용할 수 있도록 구성되었습니다.

이 가이드를 통해 개발자는 SDK의 카드 관리 기능을 자신 있게 다룰 수 있게 될 것입니다. 이제 SDK 통합의 첫 단계인 환경 설정 및 사전 확인 사항으로 넘어가겠습니다.

2.0 사전 준비: Samsung Pay 상태 확인 및 SDK 초기화

안정적인 카드 관리 기능을 구현하기 위해서는 개발을 시작하기 전에 몇 가지 필수적인 환경 설정 단계를 거쳐야 합니다. 가장 중요한 첫 단계는 사용자의 디바이스에서 삼성 페이 서비스의 가용성을 확인하는 것입니다. 이 과정을 통해 앱이 삼성 페이를 지원하지 않는 환경에서 비정상적으로 동작하는 것을 방지하고, 사용자에게 명확한 안내를 제공할 수 있습니다.

2.1 삼성 페이 상태 확인

SamsungPay 클래스는 디바이스의 삼성 페이 상태를 확인할 수 있는 getSamsungPayStatus(StatusListener listener) 메서드를 제공합니다. 이 메서드를 호출하여 현재 삼성 페이의 준비 상태를 파악해야 합니다.

StatusListener 콜백의 onSuccess 메서드는 다음과 같은 주요 상태 코드를 반환합니다. 각 상태에 따라 애플리케이션은 적절한 후속 조치를 취해야 합니다.

* SpaySdk.SPAY_READY: 삼성 페이가 활성화되어 있으며, 즉시 사용할 수 있는 상태입니다. 이 상태에서는 카드 관리 기능을 자유롭게 호출할 수 있습니다.
* SpaySdk.SPAY_NOT_SUPPORTED: 디바이스가 삼성 페이를 지원하지 않거나, 삼성 페이 앱이 설치되어 있지 않습니다. 이 경우, 사용자에게 삼성 페이를 사용할 수 없음을 안내해야 합니다.
* SpaySdk.SPAY_NOT_READY: 삼성 페이를 지원하지만, 아직 완전히 활성화되지 않은 상태입니다. 이 경우, 콜백으로 전달되는 errorData 번들에서 SpaySdk.EXTRA_ERROR_REASON 키를 확인하여 추가적인 조치를 결정해야 합니다.
  * SpaySdk.ERROR_SPAY_APP_NEED_TO_UPDATE: 삼성 페이 앱의 업데이트가 필요합니다. SamsungPay 인스턴스의 goToUpdatePage() 메서드를 호출하여 사용자를 업데이트 페이지로 안내해야 합니다.
  * SpaySdk.ERROR_SPAY_SETUP_NOT_COMPLETED: 삼성 페이 초기 설정(예: 삼성 계정 로그인)이 완료되지 않았습니다. SamsungPay 인스턴스의 activateSamsungPay() 메서드를 호출하여 사용자가 설정을 완료하도록 유도해야 합니다.
* SpaySdk.SPAY_NOT_ALLOWED_TEMPORALLY: 보안상의 이유로 삼성 페이를 일시적으로 사용할 수 없는 상태입니다. 주된 원인은 SpaySdk.ERROR_SPAY_CONNECTED_WITH_EXTERNAL_DISPLAY로, 디바이스가 외부 디스플레이에 연결된 경우입니다. 사용자에게 외부 디스플레이 연결을 해제하도록 안내해야 합니다.

// PartnerInfo를 생성합니다. (자세한 내용은 2.2절 참조)
PartnerInfo pInfo = new PartnerInfo(serviceId, bundle);
SamsungPay samsungPay = new SamsungPay(context, pInfo);

samsungPay.getSamsungPayStatus(new StatusListener() {
    @Override
    public void onSuccess(int status, Bundle data) {
        if (status == SpaySdk.SPAY_READY) {
            Log.d(TAG, "Samsung Pay is ready on the device");
            // 카드 관리 기능 호출
        } else if (status == SpaySdk.SPAY_NOT_READY) {
            int reason = data.getInt(SpaySdk.EXTRA_ERROR_REASON);
            if (reason == SpaySdk.ERROR_SPAY_APP_NEED_TO_UPDATE) {
                samsungPay.goToUpdatePage();
            } else if (reason == SpaySdk.ERROR_SPAY_SETUP_NOT_COMPLETED) {
                samsungPay.activateSamsungPay();
            }
        } else if (status == SpaySdk.SPAY_NOT_ALLOWED_TEMPORALLY) {
            int reason = data.getInt(SpaySdk.EXTRA_ERROR_REASON);
            if (reason == SpaySdk.ERROR_SPAY_CONNECTED_WITH_EXTERNAL_DISPLAY) {
                // 사용자에게 외부 디스플레이 연결 해제 안내
                Log.w(TAG, "Samsung Pay is temporarily blocked due to an external display.");
            }
        } else {
            // SPAY_NOT_SUPPORTED 또는 기타 상태
            Log.d(TAG, "Device does not support Samsung Pay");
        }
    }

    @Override
    public void onFail(int errorCode, Bundle errorData) {
        Log.e(TAG, "onFail callback is called, errorCode: " + errorCode);
    }
});


2.2 SDK 초기화: PartnerInfo 설정

CardManager를 포함한 Samsung Pay SDK의 모든 기능을 사용하려면, 파트너 정보를 담은 PartnerInfo 객체를 생성하여 SDK를 초기화해야 합니다. 이 객체는 파트너 앱을 식별하고 인증하는 데 사용됩니다.

PartnerInfo 생성자는 삼성 페이 개발자 포털에서 발급받은 serviceId와 파트너의 추가 정보를 담는 Bundle 객체를 인자로 받습니다.

* serviceId: 파트너를 고유하게 식별하는 ID입니다.
* Bundle: 서비스 유형(PARTNER_SERVICE_TYPE)이나 발급사 이름(EXTRA_ISSUER_NAME)과 같은 추가 메타데이터를 설정하는 데 사용됩니다.

// 삼성 페이 개발자 포털에서 발급받은 서비스 ID
String serviceId = "sampleServiceId";
String issuerName = "MyBank";

Bundle bundle = new Bundle();
// 파트너 서비스 유형 설정 (예: App-to-App)
bundle.putString(SpaySdk.PARTNER_SERVICE_TYPE, SpaySdk.ServiceType.APP2APP.toString());
// 발급사 이름 설정
bundle.putString(SpaySdk.EXTRA_ISSUER_NAME, issuerName);

// PartnerInfo 객체 생성
PartnerInfo pInfo = new PartnerInfo(serviceId, bundle);


삼성 페이 상태 확인 및 SDK 초기화가 완료되었으므로, 이제 카드 관리의 핵심인 CardManager 클래스를 본격적으로 살펴보겠습니다.

3.0 핵심 클래스 분석: CardManager

Samsung Pay SDK에서 CardManager는 모든 카드 관련 API 호출의 시작점 역할을 하는 핵심 클래스입니다. 이 클래스는 삼성 페이에 등록된 카드를 관리하기 위한 API를 제공하며, 카드 추가, 조회, 인증 등의 기능을 수행합니다. 따라서 이 클래스를 올바르게 인스턴스화하고 사용하는 것이 카드 관련 기능을 구현하는 데 매우 중요합니다.

CardManager 인스턴스는 다음과 같이 new CardManager(android.content.Context context, PartnerInfo partnerInfo) 생성자를 통해 생성합니다. 이 때, 2.2절에서 생성한 PartnerInfo 객체를 전달해야 합니다.

Context ct = activity; // 또는 Service Context
// 이전 단계에서 생성한 PartnerInfo 객체
PartnerInfo pInfo = new PartnerInfo(serviceId, bundle);

// CardManager 인스턴스 생성
CardManager cardManager = new CardManager(ct, pInfo);


중요 참고사항: 인스턴스 관리 정책

새로운 CardManager 인스턴스가 생성될 때마다, 이전에 다른 인스턴스에서 보낸 모든 보류 중인 비동기 요청(addCard, getAllCards 등)이 즉시 취소됩니다. 이로 인해 콜백이 전혀 호출되지 않거나, 예측 불가능한 동작 및 경합 상태(Race Condition)가 발생할 수 있습니다.

따라서 안정적인 애플리케이션 아키텍처를 위해, 파트너 애플리케이션의 단일 세션(예: 하나의 Activity 생명주기) 내에서는 하나의 CardManager 인스턴스만 생성하고 유지하여 사용하는 것이 매우 중요한 아키텍처 모범 사례입니다. 이는 불필요한 요청 취소를 방지하고 API 호출의 일관성과 안정성을 보장합니다.

CardManager가 준비되었으니, 이를 활용하여 첫 번째 핵심 기능인 카드 추가 프로세스를 구현해 보겠습니다.

4.0 카드 관리 기능 구현: 단계별 가이드

이제 CardManager를 사용하여 실제 카드 관리 기능을 구현하는 구체적인 방법을 단계별로 안내합니다. 카드 추가, 조회, 인증 기능은 각각 독립적이면서도 상호 보완적인 비즈니스 로직을 구성합니다. 예를 들어, 카드를 추가한 후에는 해당 카드의 상태를 조회해야 할 수 있으며, 경우에 따라 추가 인증 절차를 거쳐야 합니다. 각 기능의 중요성과 상호작용을 이해하며 따라오시기 바랍니다.

4.1 카드 추가하기 (addCard)

addCard 메서드는 파트너 앱(예: 은행 앱)에서 사용자의 카드 정보를 삼성 페이로 직접 전달하여 등록하는 데 사용됩니다. 이를 통해 사용자는 앱을 전환하지 않고도 간편하게 카드를 추가할 수 있습니다.

void addCard(@NonNull AddCardInfo addCardInfo, @NonNull AddCardListener listener)


1. AddCardInfo 객체 생성

먼저, 추가할 카드의 상세 정보를 담은 AddCardInfo 객체를 생성해야 합니다. 여기에는 카드 유형, 토큰화 제공자, 그리고 프로비저닝 페이로드(Provisioning Payload)가 포함됩니다.

프로비저닝 페이로드는 Visa, Mastercard 등 카드 네트워크의 사양에 따라 생성되는 데이터 문자열입니다. Samsung Pay SDK는 이 페이로드를 위한 전송 메커니즘 역할만 하며, 그 내용을 정의하거나 해석하지 않습니다. 올바른 데이터를 생성하려면 반드시 해당 카드 네트워크의 문서를 참조해야 합니다.

String cardType = Card.CARD_TYPE_CREDIT; // 카드 유형 (신용카드)
String tokenizationProvider = AddCardInfo.PROVIDER_ABCD; // 토큰화 제공자

// 카드 네트워크 규격에 따라 생성된 프로비저닝 페이로드
String testPayload = "ThisIsTestPayloadCardInfo1234567890";

Bundle cardDetail = new Bundle();
cardDetail.putString(AddCardInfo.EXTRA_PROVISION_PAYLOAD, testPayload);

AddCardInfo addCardInfo = new AddCardInfo(cardType, tokenizationProvider, cardDetail);


2. addCard 호출 및 결과 처리

생성된 AddCardInfo 객체를 사용하여 addCard 메서드를 호출하고, AddCardListener 콜백을 통해 결과를 처리합니다.

* onSuccess(int status, Card card): 카드 추가에 성공하면 호출됩니다. 성공적으로 추가된 Card 객체가 반환됩니다.
* onFail(int errorCode, Bundle errorData): 카드 추가에 실패하면 호출됩니다. errorData 번들 내의 SpaySdk.EXTRA_ERROR_REASON 키를 통해 실패의 구체적인 원인을 파악할 수 있습니다.
* onProgress(int currentCount, int totalCount, Bundle bundleData): TSM(Trusted Service Manager) 솔루션과 같이 카드 다운로드에 시간이 걸리는 경우, 진행 상황을 보고하기 위해 호출될 수 있습니다.

cardManager.addCard(addCardInfo, new AddCardListener() {
    @Override
    public void onSuccess(int status, Card card) {
        Log.d(TAG, "onSuccess callback is called. Card ID: " + card.getCardId());
    }

    @Override
    public void onFail(int errorCode, Bundle errorData ) {
        Log.e(TAG, "onFail callback is called, errorCode: " + errorCode);
        if (errorData != null) {
            int reason = errorData.getInt(SpaySdk.EXTRA_ERROR_REASON);
            Log.e(TAG, "Failure Reason: " + reason);
        }
    }

    @Override
    public void onProgress(int currentCount, int totalCount, Bundle bundleData) {
        Log.d(TAG, "onProgress callback is called: " + currentCount + " / " + totalCount);
        // TSM 기반 카드 다운로드 진행률 UI 업데이트 등
    }
});


4.2 등록된 카드 목록 조회하기 (getAllCards)

getAllCards 메서드는 현재 삼성 페이에 등록된 카드 목록을 조회하는 데 사용됩니다. 파트너사는 이 API를 통해 자사에서 발급한 카드 목록을 사용자에게 보여주거나 관리할 수 있습니다.

void getAllCards(@Nullable android.os.Bundle cardFilter, @NonNull GetCardListener listener)


cardFilter 파라미터는 현재 코드 상에서는 null로 전달할 수 있습니다. 파트너 앱이 자사에서 발급한 카드만 조회하도록 하는 필터링 로직은 Samsung Pay 개발자 포털에서 온보딩 과정 중에 발급사 이름을 정의함으로써 서버 측에서 처리됩니다. 따라서 개발자가 앱 코드 내에서 복잡한 클라이언트 측 필터링을 구현할 필요가 없습니다.

1. getAllCards 호출 및 결과 처리

GetCardListener의 onSuccess(List<Card> cards) 콜백을 통해 조회된 Card 객체 리스트를 수신합니다. 리스트가 비어있지 않다면, 각 Card 객체를 순회하며 필요한 정보를 추출할 수 있습니다.

cardManager.getAllCards(null, new GetCardListener() {
    @Override
    public void onSuccess(List<Card> cards) {
        if (cards == null || cards.isEmpty()) {
            Log.e(TAG,"No card is found");
            return;
        }

        // 조회된 카드 목록 처리
        for (Card card : cards) {
            Log.d(TAG, "CardId: " + card.getCardId() + ", CardStatus: " + card.getCardStatus());

            // getCardInfo()를 통해 추가 정보 추출
            if (card.getCardInfo() != null) {
                Bundle cardInfo = card.getCardInfo();
                String last4FPan = cardInfo.getString(SpaySdk.EXTRA_LAST4_FPAN);
                String last4DPan = cardInfo.getString(SpaySdk.EXTRA_LAST4_DPAN);
                String cardIssuerName = cardInfo.getString(SpaySdk.EXTRA_ISSUER_NAME);

                Log.d(TAG, "Last 4 FPAN: " + last4FPan + ", Last 4 DPAN: " + last4DPan);
            }
        }
    }

    @Override
    public void onFail(int errorCode, Bundle errorData) {
        Log.e(TAG, "onFail callback is called, errorCode: " + errorCode);
    }
});


card.getCardInfo() 메서드는 추가 정보를 담은 Bundle을 반환하며, 이를 통해 카드의 FPAN 마지막 4자리(EXTRA_LAST4_FPAN), DPAN 마지막 4자리(EXTRA_LAST4_DPAN), 발급사 이름(EXTRA_ISSUER_NAME) 등의 유용한 정보를 얻을 수 있습니다.

4.3 카드 인증 및 활성화하기 (verifyCardIdv)

카드를 추가하는 과정에서 발급사나 카드 네트워크가 2차 인증(IDV, Identity Verification)을 요구할 수 있습니다. verifyCardIdv 메서드는 사용자가 파트너 앱을 통해 인증을 완료한 후, 해당 인증 정보를 삼성 페이로 전달하여 카드를 최종적으로 활성화하는 데 사용됩니다.

void verifyCardIdv(@NonNull IdvVerifyInfo idvVerifyInfo, @NonNull StatusListener listener)


1. IdvVerifyInfo 객체 생성

카드 ID, 서버로부터 받은 인증 코드, 그리고 기타 필요한 정보를 담아 IdvVerifyInfo 객체를 생성합니다. getAllCards를 통해 조회한 카드가 PENDING_PROVISION 상태일 때 이 과정이 필요할 수 있습니다.

// getAllCards() API를 통해 얻은 카드 ID
String cardId = "card_id_example_dummy";
// 발급사 서버로부터 받은 인증 코드
String authCode = "auth_code_example_dummy";

// 카드 네트워크/발급사 서버에 전달할 추가 정보 (선택 사항)
Bundle cardInfoData = new Bundle();

IdvVerifyInfo idvVerifyInfo = new IdvVerifyInfo(cardId, authCode, cardInfoData);

// 인증 유형 설정 (기본값: IDV_TYPE_APP2APP)
idvVerifyInfo.setIDnVType(IdvVerifyInfo.IDV_TYPE_APP2APP);


2. verifyCardIdv 호출 및 결과 처리

생성된 IdvVerifyInfo 객체로 메서드를 호출하고, StatusListener를 통해 인증 요청의 성공 또는 실패 결과를 처리합니다.

cardManager.verifyCardIdv(idvVerifyInfo, new StatusListener() {
    @Override
    public void onSuccess(int status, Bundle data) {
        Log.d(TAG, "Verification code successfully sent to server");
        // 카드 활성화 성공 후 로직 처리
    }

    @Override
    public void onFail(int errorCode, Bundle errorData) {
        Log.e(TAG, "onFail callback is called, errorCode: " + errorCode);
        // 인증 실패 처리
    }
});


이제 카드 관리의 핵심 기능들을 구현했으므로, API 호출 결과로 반환되는 Card 객체의 상태와 주요 오류 코드를 이해하는 것이 중요합니다.

5.0 핵심 데이터 객체 이해: Card 상태

CardManager API를 통해 반환되는 Card 객체는 삼성 페이에 등록된 카드의 모든 정보를 담고 있는 핵심 데이터 모델입니다. 카드의 현재 상태를 정확히 파악하는 것은 사용자에게 올바른 피드백을 제공하고, 비즈니스 로직에 따른 후속 조치를 결정하는 데 매우 중요합니다.

Card 클래스는 다음과 같은 주요 메서드를 제공합니다.

* getCardId(): 삼성 페이 내에서 카드를 고유하게 식별하는 ID를 반환합니다.
* getCardStatus(): 카드의 현재 상태(예: 활성, 정지 등)를 문자열로 반환합니다.
* getCardBrand(): 카드 브랜드를 반환합니다.
* getCardInfo(): 카드의 부가 정보(FPAN/DPAN 마지막 4자리 등)를 담은 Bundle을 반환합니다.

특히 getCardStatus()가 반환하는 상태 값은 애플리케이션의 흐름을 제어하는 데 결정적인 역할을 합니다. 주요 카드 상태와 그 의미는 다음과 같습니다.

상태 (Status)	설명
ACTIVE	카드가 삼성 페이에 완전히 등록되어 활성화된 상태입니다.
PENDING_PROVISION	사용자가 2차 인증(IDV)을 완료하지 않아 카드가 완전히 등록되지 않은 상태입니다.
PENDING_ACTIVATION	카드가 등록되었지만 아직 활성화되지 않은 상태입니다.
SUSPENDED	발급사, 카드 네트워크 또는 사용자에 의해 사용이 일시 중단된 상태입니다.
DISPOSED	카드가 삼성 페이에서 삭제된 상태입니다.
EXPIRED	카드의 유효 기간이 만료된 상태입니다.

이처럼 카드의 상태를 이해하는 것은 안정적인 애플리케이션 로직을 구축하는 데 필수적이며, 다음으로 살펴볼 오류 처리 전략의 기반이 됩니다.

실용적인 적용

만약 getAllCards가 PENDING_PROVISION 상태의 카드를 반환했다면, 애플리케이션 로직은 사용자에게 필요한 인증을 완료하도록 안내해야 합니다. 사용자가 OTP(일회용 비밀번호)와 같은 필수 정보를 제공하면, 이 정보를 사용하여 verifyCardIdv 메서드를 호출하여 카드를 활성화해야 합니다. 이 워크플로우는 2차 인증 요구사항을 처리하는 데 매우 중요합니다.

6.0 필수 전략: 오류 처리

안정적이고 사용자 친화적인 애플리케이션을 개발하기 위해서는 예외 상황을 효과적으로 처리하는 것이 매우 중요합니다. Samsung Pay SDK는 API 호출 실패 시 다양한 오류 코드를 반환하며, 개발자는 이 코드를 이해하고 각 상황에 맞는 적절한 대응 방안을 구현해야 합니다.

CardManager의 API 호출 실패 시, onFail 콜백을 통해 오류 코드가 전달됩니다. CardManager.PUSH_PROVISION_ERROR 테이블과 AppToAppConstants 인터페이스에 정의된 주요 오류 코드는 다음과 같습니다.

* ERROR_CARD_ALREADY_REGISTERED: 이미 삼성 페이에 등록된 카드 추가를 시도할 때 발생합니다. 사용자에게 해당 카드가 이미 등록되어 있음을 안내해야 합니다.
* ERROR_INVALID_CARD / ERROR_INVALID_CARDINPUT: 카드 정보가 유효하지 않거나 사용자가 잘못된 정보를 입력했을 때 발생합니다. 사용자에게 입력 정보를 다시 확인하도록 요청해야 합니다.
* ERROR_MAX_CARD_NUM_REACHED: 삼성 페이에 등록할 수 있는 최대 카드 개수(디바이스 한도)에 도달했을 때 발생합니다.
* ERROR_CARD_NOT_SUPPORTED: 해당 카드가 삼성 페이에서 지원되지 않음을 나타냅니다.
* ERROR_SERVER_REJECT: 발급사 또는 카드 네트워크 서버가 요청을 거부했을 때 발생합니다.
* ERROR_AUTH_CODE_INVALID / ERROR_AUTH_CODE_EXPIRED: verifyCardIdv 호출 시 사용된 인증 코드가 유효하지 않거나 만료되었을 때 발생합니다. 사용자에게 인증 코드를 다시 받도록 안내해야 합니다.

onFail(int errorCode, Bundle errorData) 콜백의 errorData 번들은 오류 해결에 중요한 추가 정보를 제공합니다.

모범 사례: 항상 errorCode와 errorData 번들 전체를 로깅하십시오. 특히 SpaySdk.EXTRA_ERROR_REASON과 SpaySdk.EXTRA_REQUEST_ID는 매우 중요합니다. EXTRA_REQUEST_ID는 Samsung 개발자 지원팀과 소통하여 서버 측 문제를 디버깅할 때 필수적인 정보입니다.

@Override
public void onFail(int errorCode, Bundle errorData) {
    Log.e(TAG, "onFail callback is called, errorCode: " + errorCode);
    if (errorData != null) {
        // 더 구체적인 오류 원인 확인
        int reason = errorData.getInt(SpaySdk.EXTRA_ERROR_REASON);
        // 디버깅을 위한 요청 ID 확인
        String requestId = errorData.getString(SpaySdk.EXTRA_REQUEST_ID);

        Log.e(TAG, "Extra Error Reason: " + reason + ", Request ID: " + requestId);
        // reason 코드에 따른 사용자 안내 로직 구현
    }
}


이 가이드를 통해 삼성 페이 SDK의 카드 관리 기능 구현에 대한 기본적인 이해를 마쳤습니다.
